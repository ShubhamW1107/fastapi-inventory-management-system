# inventory-management-system
1️⃣ Headline<br>
A full-stack Inventory Management System built using FastAPI, SQLAlchemy, Microsoft SQL Server, HTML, CSS, JavaScript, and Bootstrap. The application provides a responsive web interface and RESTful APIs to efficiently manage product inventory through complete CRUD operations.
2️⃣ Purpose<br>
The Inventory Management System is designed to simplify product inventory management by providing a centralized platform to add, update, view, search, and delete products.<br>
The project demonstrates backend API development using FastAPI, database integration with Microsoft SQL Server, and frontend interaction through a responsive web interface.<br>
It was built as a learning project to gain hands-on experience in developing REST APIs, database connectivity, and full-stack application development.<br>
3️⃣ Tech Stack<br>
The project was built using the following technologies:<br>
🐍 Python – Core programming language used for backend development.<br>
⚡ FastAPI – Framework used to build high-performance REST APIs.<br>
🗄️ Microsoft SQL Server – Relational database used to store product information.<br>
🔗 SQLAlchemy ORM – Used for database modeling and CRUD operations.<br>
✅ Pydantic – Used for request validation and response serialization.<br>
🚀 Uvicorn – ASGI server used to run the FastAPI application.<br>
🌐 HTML5 – Developed the frontend user interface.<br>
🎨 CSS3 – Styled the application with a modern layout.<br>
💻 JavaScript – Added client-side functionality and API interaction.<br>
🅱️ Bootstrap 5 – Used for responsive UI components and layout.<br>
📖 Swagger UI – Used for automatic API documentation and endpoint testing.<br>
4️⃣ Data Source<br>
The application uses Microsoft SQL Server as the backend database.<br>
Each product record contains:<br>
Product ID<br>
Product Name<br>
Category<br>
Description<br>
Price<br>
Quantity<br>
The data is managed through REST APIs developed using FastAPI and stored permanently in SQL Server using SQLAlchemy ORM.<br>
5️⃣ Features<br>
Business Problem<br>
Many small businesses manage inventory manually using spreadsheets or paper records, which can lead to:<br>
1) Duplicate product entries<br>
2) Difficulty updating inventory<br>
3) Time-consuming product searches<br>
4) Poor inventory tracking<br>
5) Increased chances of human error<br>
Managing products through a centralized web application improves efficiency and simplifies inventory operations.<br>
Goal of the Project<br>
The objective of this project is to build a simple and user-friendly Inventory Management System that:<br>
1) Allows users to manage product records efficiently.<br>
2) Performs complete CRUD operations using REST APIs.<br>
3) Stores inventory data securely in Microsoft SQL Server.<br>
4) Demonstrates backend development using FastAPI.<br>
5) Provides an easy-to-use responsive web interface.<br>
6) Enables API testing through Swagger UI.<br>
